# Shared Briefing, Digest, and Summary Surface Standard for the Fourth Form Retail Decision Intelligence Platform

## Purpose of This Document

This document defines the shared platform standard for briefing surfaces, digest surfaces, and summary surfaces across all current and future domains.

It exists because the platform now has governed standards for cases, recommendations, rationale, evidence, uncertainty, constraints, action paths, urgency, review packets, review resolution, commitment boundaries, execution outcomes, output packages, post-mortem judgment, and policy-learning admission, but it still lacks one shared meaning for the summarization layer that helps different audiences understand those objects without replacing them, collapsing them, or silently rewriting their governed meaning.

Without a shared standard, the platform will drift into domain-specific briefing habits, executive summaries that silently strip uncertainty or authority context, operator digests that hide action-path ambiguity, reviewer surfaces that collapse packet meaning into presentation notes, post-mortem summaries that blur attribution into narrative shorthand, learning digests that look like admitted evidence before they actually are, and presentation-safe abstractions that erase the very qualifiers later decision-loop stages still need.

This document is therefore a control document for shared briefing, digest, and summary surface structure.

It defines the core concepts, shared surface meanings, audience and compression rules, omission and traceability rules, interaction rules, extension rules, and governance linkage that all domains must follow when producing governed summarization surfaces above underlying canonical objects.

It is the canonical shared briefing, digest, and summary surface standard for the platform. Future domain workflow contracts, review surfaces, reporting surfaces, post-mortem communications, learning handoff surfaces, and other audience-specific summarization layers must align with it unless a formal decision record explicitly revises it.

## Role of This Document in the Platform

This document governs the shared summarization layer that sits above underlying controlled objects and below audience use.

The shared decision case and decision memory standard defines the governed episode anchor and reusable memory. The shared recommendation record standard defines what the platform recommended. The shared decision rationale and explanation trace standard defines the internal rationale trace and the governed explanation derivation that may safely expose parts of that rationale. The shared evidence bundle and signal provenance standard defines what counted as evidence and where it came from. The shared uncertainty and confidence context standard defines what qualified decision strength. The shared constraint and feasibility context standard defines what bounded valid action. The shared action-path and candidate-action-set standard defines what serious alternatives existed. The shared decision materiality, priority, and urgency standard defines what significance and timing posture applied. The shared human-review-packet and intervention-handoff standard defines the governed assembly for accountable human intervention. The shared review resolution and case disposition standard defines how review concluded. The shared recommendation, commitment, and action-instruction boundary standard defines how advisory output differs from binding commitment and executable instruction. The shared execution deviation and outcome standard defines what later happened in reality. The shared post-mortem standard defines the governed attribution object and its learning direction. The shared output-package and scope metadata standard defines how packages carry scope, lineage, and entitlement metadata. This document governs the derived briefing, digest, and summary surfaces that may present those objects differently for different audiences without changing the underlying canonical meaning.

In practical terms, this document governs what a briefing surface is, what a digest surface is, what a summary surface is, how those surfaces differ from recommendation, rationale, packet, output package, and post-mortem objects, how audience-specific summarization may compress detail without severing traceability, how internal full-fidelity and presentation-safe layers differ, what omission is allowed, what must remain linkable, and how later decision-loop stages should treat these surfaces without mistaking them for the underlying source of truth.

This document therefore governs governed summarization and briefing structure as part of platform coherence.

## Core Thesis

In the Fourth Form platform, briefing surfaces, digest surfaces, and summary surfaces must remain first-class governed summarization layers whose audience fit, compression boundary, omission risk, traceability, sufficiency, scope discipline, and lineage remain explicit enough that the platform can help humans understand a case, a review path, an exception state, a post-mortem, or a learning candidate without turning those audience-facing surfaces into semantic replacements for the underlying canonical objects.

That is the core thesis.

A summary surface is not the same thing as a recommendation. A digest is not the same thing as a rationale trace. A human-review packet is not the same thing as a briefing surface. Compression must not become semantic rewrite. Presentation-safe briefing is not the same thing as internal full-fidelity briefing. Omission risk must remain explicit. Summary sufficiency is not the same thing as decision sufficiency. A post-mortem briefing is not the same thing as a post-mortem object. A learning digest is not the same thing as policy-learning admission. Future summary extensions must be placed according to control role, not convenience.

## What This Standard Is and Is Not

This standard is the shared platform rule for how the system records, links, preserves, and reuses governed briefing surfaces, digest surfaces, and summary surfaces.

It is not a UI note. It is not a reporting template. It is not a product copy guide. It is not a local workflow memo. It is not an output-package schema. It is not a recommendation record. It is not the rationale trace. It is not a human-review packet. It is not a review-resolution object. It is not a post-mortem object. It is not a policy-learning admission record. It is not permission for domains to rewrite controlled object meaning in lighter prose and later treat that prose as if it were the governing source of truth. It is not permission to compress away uncertainty, authority, failure, attribution, or traceability merely because an audience prefers shorter text.

A real shared briefing, digest, and summary surface standard means the platform can answer the following questions for any summarization surface: what underlying objects the surface derives from, what audience and purpose the surface serves, what compression boundary governed the surface, what was omitted or compressed, what omission risk remains active, whether the surface is presentation-safe or internal full-fidelity, whether the surface is sufficient for its audience purpose, and how a reader can trace the surface back to the underlying controlled objects without ambiguity.

## Why a Briefing, Digest, and Summary Surface Standard Is Necessary

Domains must not define briefing, digest, and summary semantics independently because the platform cannot remain one governed decision system if one domain treats an executive summary as if it were the recommendation itself, another treats a rationale digest as if it were the full rationale trace, another treats packet excerpts as if they were a review packet, another strips uncertainty for executives but leaves action pressure intact, another compresses post-mortem attribution into an outcome headline, and another uses a learning digest as if it were already admitted evidence for policy change.

If briefing and summary grammar is left local, several failures follow. Readers stop being able to tell the difference between a controlled object and a derived surface. Presentation-safe layers start muting material qualifiers that downstream audiences still need. Review and exception surfaces begin hiding authority or anomaly context because those details feel operationally inconvenient. Summaries become shorter by erasing linkage instead of by compressing responsibly. Digest reuse starts drifting into semantic substitution, and later execution review, post-mortem, or policy-learning analysis inherits thin surfaces whose omissions were never governed or even made visible.

The platform therefore needs one shared standard so that future domains can extend one governed summarization grammar rather than inventing local meanings for briefings, digests, summaries, omission, compression, audience fit, and traceability.

## Core Concepts

The platform uses the following core concepts.

### Briefing surface

A briefing surface is the richest governed summarization layer for a defined audience and purpose, designed to help a human understand the materially relevant shape of a case or later decision-loop artifact without replacing the underlying controlled objects.

### Digest surface

A digest surface is a governed intermediate summary layer that compresses a larger briefing or object set into a tighter form while preserving the minimum traceability, qualifiers, and linkage needed for its intended audience and purpose.

### Summary surface

A summary surface is the smallest governed abstraction layer that still preserves traceable linkage back to the underlying controlled objects and still communicates the materially relevant point for its audience and purpose.

### Audience-specific briefing

Audience-specific briefing is a briefing surface whose included detail, terminology, and presentation-safe filtering are deliberately shaped for a specific audience while preserving underlying object meaning.

### Internal full-fidelity briefing

Internal full-fidelity briefing is the richest internally authorized briefing layer, intended for audiences permitted to see materially complete rationale, evidence, uncertainty, authority, failure, and lineage context without presentation-safe reduction.

### Presentation-safe briefing

Presentation-safe briefing is a governed briefing layer derived from the same underlying objects but filtered, redacted, or compressed to match authorized reporting scope, role-sensitive access, and delivery constraints.

### Compression boundary

Compression boundary is the governed limit beyond which a summarization surface would cease to preserve the materially relevant meaning, qualifiers, or linkage required for its intended audience and purpose.

### Omission risk

Omission risk is the governed risk that compressed or filtered presentation may leave out context whose absence could materially distort audience understanding, downstream handling, or later reuse.

### Summary traceability

Summary traceability is the governed ability to move from a briefing, digest, or summary surface back to the source case, recommendation, rationale, evidence, packet, review, execution, post-mortem, or learning objects from which it was derived.

### Summary sufficiency

Summary sufficiency is the governed judgment that a summary surface is adequate for its intended audience and purpose because it preserves the minimum required meaning, qualifiers, and traceability without pretending to be the underlying objects.

### Briefing lineage

Briefing lineage is the reconstructible chain connecting a briefing surface to the source objects, transformations, audience filters, and later uses that depended on it.

### Digest lineage

Digest lineage is the reconstructible chain connecting a digest surface to the richer briefing or source objects from which it was compressed and to any later surfaces derived from it.

### Summary lineage

Summary lineage is the reconstructible chain connecting a summary surface to the governed source objects, the compression decisions that shaped it, and the later workflows or audiences that consumed it.

### Summary distortion risk

Summary distortion risk is the governed risk that compression, omission, or audience-fitting will change perceived meaning strongly enough that the surface no longer truthfully reflects the underlying canonical objects.

### Audience-fit without semantic rewrite

Audience-fit without semantic rewrite is the governed rule that a summarization surface may adjust detail level, order, and presentation emphasis for a particular audience, but may not change what the underlying objects mean.

## Shared Briefing Surface

At platform level, a shared briefing surface is the formal governed summarization layer that presents the materially relevant shape of a case or related decision-loop artifact in a way that helps a human understand it quickly while remaining anchored to the underlying controlled objects.

It exists because many audiences need something more navigable than the full object graph while still needing more context than a thin summary can safely provide. The platform therefore needs a surface that can organize recommendation, rationale, evidence, uncertainty, constraints, action paths, urgency, authority, review, execution, post-mortem, or learning context into an intelligible derived layer without implying that the briefing itself is now the canonical object.

The shared briefing surface must preserve, conceptually, all of the following. It must preserve a stable briefing identity so the surface is reconstructible. It must preserve audience and purpose reference so later systems can tell why the surface was produced and for whom. It must preserve source-object linkage so readers can trace the surface back to the relevant case, recommendation, rationale, evidence, packet, review, post-mortem, or learning objects. It must preserve compression and omission posture so the reader can tell whether the surface is internal full-fidelity or presentation-safe and what categories of detail were compressed. It must preserve the materially relevant qualifiers for its audience, including uncertainty, authority, urgency, failure, or attribution context where those are material to safe understanding. It must preserve lineage or version reference and timestamp so later systems can reconstruct what briefing existed at the relevant time.

The briefing surface is the richest governed summarization layer. It helps a human understand the case, but it does not replace the underlying canonical objects. A human-review packet is not the same thing as a briefing surface. A briefing surface may reference packet content, but packet sufficiency, packet readiness, and intervention actionability remain controlled by the packet standard.

Internal full-fidelity briefing may preserve materially complete rationale, evidence conflict, uncertainty, action-path alternatives, authority posture, anomaly state, review posture, or attribution basis where the audience is entitled to see them. Presentation-safe briefing may compress or filter some of those details for scope, entitlement, or delivery reasons, but presentation-safe briefing is not the same thing as internal full-fidelity briefing. Presentation-safe filtering may reduce exposure, not change meaning.

## Shared Digest Surface

At platform level, a shared digest surface is the formal governed intermediate summary layer that compresses a richer briefing or richer object set into a more compact but still governed form.

It exists because many audiences do not need the full richness of a briefing surface for every interaction, but they still need more than a headline. The platform therefore needs a controlled intermediate layer that can compress without severing lineage, compress without erasing qualifiers that remain material, and compress without turning itself into an untracked local memo.

The shared digest surface must preserve, conceptually, all of the following. It must preserve stable digest identity so the surface remains reconstructible. It must preserve audience and purpose reference so the degree of compression is intelligible. It must preserve source linkage to the richer briefing or directly to the source objects where no prior briefing existed. It must preserve omission and compression posture so later systems can tell what categories of detail were intentionally collapsed. It must preserve the minimum qualifiers necessary to keep the digest honest for its purpose, including uncertainty, authority, urgency, failure, attribution, or action-path clarity when those remain materially relevant. It must preserve digest lineage, timestamp, and version reference so later readers can reconnect the digest to what it compressed.

A digest is not the same thing as a rationale trace. The rationale trace preserves the authoritative internal reasoning structure. The digest may summarize the central thesis, the main supporting and weakening context, or the principal path comparison, but it must not claim to be the rationale trace itself or silently flatten conflicting rationale into a cleaner story.

The digest is therefore a governed intermediate surface, not merely a shorter paragraph. It is intended to be compact, but not semantically free.

## Shared Summary Surface

At platform level, a shared summary surface is the formal governed smallest abstraction layer that still preserves traceable linkage to the underlying controlled objects.

It exists because some audiences and moments require a minimal surface that communicates the central point quickly. The platform still needs that minimal layer to remain governed, which means the surface must preserve enough linkage, qualification, and omission visibility that it cannot be mistaken for an unsupported slogan, an untraceable executive headline, or a rewritten decision object.

The shared summary surface must preserve, conceptually, all of the following. It must preserve stable summary identity so the minimal abstraction remains referenceable. It must preserve audience and purpose reference so summary sufficiency is judged in context. It must preserve source-object linkage so the summary remains anchored to the underlying case, recommendation, rationale, packet, review, execution, post-mortem, or learning objects. It must preserve omission-risk posture so readers know when materially relevant detail has been compressed away. It must preserve the minimum qualifiers required to avoid misleading its audience. It must preserve summary lineage, timestamp, and version reference so later systems can reconstruct what minimal abstraction was used and what it derived from.

The summary surface is the smallest governed abstraction layer, but it is not permission to sever traceability. A summary surface is not the same thing as a recommendation, not the same thing as a packet, not the same thing as a post-mortem object, and not the same thing as a learning-admission record. It is a derived and traceable surface above them.

## Audience and Compression Rules

Audience-specific summarization must always preserve audience-fit without semantic rewrite. The governing question is not whether a surface is short or long. The governing question is whether the audience receives a truthful, traceable, purpose-fit derivation of the underlying objects.

### Executive audience

Executive briefing may compress implementation detail, full evidence listings, long alternative-path analysis, and secondary operational nuance when those details are not materially necessary for executive judgment. It must not silently drop material uncertainty, material authority context, material failure or anomaly context, material urgency posture, or material downside qualification. Executive summaries must not silently strip uncertainty, authority, or failure context where materially relevant.

### Operator audience

Operator briefing or digest may compress deep attribution history, broad comparative commentary, and extended rationale prose where the operator primarily needs current handling clarity. It must not silently drop current action-path clarity, constraint posture, urgency posture, execution-relevant qualifiers, or active exception or failure context. Operator digest with no action-path clarity is a governed failure.

### Reviewer audience

Reviewer briefing may compress presentational polish, lengthy duplicated explanation, and non-material delivery formatting. It must not silently drop authority context, rationale qualification, evidence conflict, uncertainty qualification, or packet-adjacent context where accountable review depends on them. Review briefings remain derived surfaces and must not pretend to be packet sufficiency by themselves.

### Exception or failure audience

Exception or failure briefing may compress broader commercial narrative and secondary business context where the primary need is structural safety, containment, anomaly interpretation, or recovery path. It must not silently drop anomaly state, failure state, integrity risk, safe-to-wait versus unsafe-to-wait posture, recovery posture, or authority path. Failure and exception briefings must remain distinguishable from normal operator summaries.

### Post-mortem audience

Post-mortem briefing may compress live operational chatter, duplicated recommendation detail, or lower-signal contextual detail that does not materially affect attribution. It must not silently drop expected-versus-realized comparison, execution-deviation posture, override relevance where present, attribution qualification, or evidence-quality qualification. A post-mortem briefing is not the same thing as a post-mortem object.

### Learning audience

Learning briefing or digest may compress presentation detail, operator wording, and non-material narrative description where the primary need is learning relevance. It must not silently drop scope validity, comparability limits, evidence-quality limits, attribution strength, omission risk, or admission status. A learning digest is not the same thing as policy-learning admission.

Across all audiences, what may be compressed is audience-relative, but what cannot be silently dropped is whatever would materially change safe interpretation, downstream handling, or reuse. Compression is allowed. Silent semantic weakening is not.

## Omission, Compression, and Traceability Rules

A compressed summary must still remain linked. Every governed briefing, digest, and summary surface must preserve traceable linkage back to the underlying controlled objects from which it was derived. Compression may shorten that path for readers, but it must not remove it.

Omission must be governed. A surface may omit detail only when the omission falls inside the allowed compression boundary for the relevant audience and purpose. Omission cannot be treated as a local stylistic preference. If the omission changes the safe interpretation of the surface, that omission is no longer safe.

Omission risk must remain explicit. Where materially relevant context has been compressed away, the surface must preserve that some omission risk remains and preserve the path back to the fuller source objects. Omission risk must remain explicit.

Compression must not become semantic rewrite. Compression may reduce length, reorder emphasis, or collapse secondary detail, but it must not change what recommendation, rationale, packet, review status, failure state, post-mortem judgment, or learning status actually mean.

Presentation-safe abstraction must not erase critical downstream meaning. Presentation-safe filtering may suppress unauthorized detail, but it must not make a safe-to-wait case look urgent, make an unresolved review look settled, make an advisory recommendation look binding, make a failure briefing look like normal operation, or make immature learning material look admitted.

Summary sufficiency is audience-relative but bounded by traceability. A surface may be sufficient for one audience purpose while being insufficient for another, but summary sufficiency is not the same thing as decision sufficiency. A concise executive surface may be sufficient for executive awareness and still be insufficient for review, commitment, or learning admission. The boundary is that sufficiency must be judged relative to audience purpose while remaining anchored to traceability and omission visibility.

What may be omitted safely includes non-material formatting detail, repeated explanatory phrasing, secondary evidence listings where the governing qualifier is still preserved, long alternative-path inventories where the central path distinction remains honest, and deep background context that is not material for the audience purpose. What must remain linkable includes the source case, the controlling object references, the materially relevant qualifiers, the applicable authority or failure context where relevant, the current action or review posture where relevant, and the governed path back to fuller context.

## Interaction with Recommendation, Rationale, Packet, and Output Package

A summary surface is not the same thing as a recommendation. Recommendation meaning remains controlled by the recommendation standard. A briefing, digest, or summary may describe or reference the recommendation, but it must not be treated as though it were the recommendation record itself.

A digest is not the same thing as a rationale trace. The digest may summarize the main thesis, supporting logic, or central trade-off, but the authoritative internal reasoning structure remains with the rationale trace and its controlled explanation derivations.

A human-review packet is not the same thing as a briefing surface. The packet governs accountable intervention sufficiency and handoff readiness. A briefing surface may assist understanding of a case that also has a packet, but the briefing surface does not by itself satisfy packet sufficiency, packet readiness, or downstream intervention readiness.

A summary is not the output package itself. The output package governs delivery, scope metadata, entitlement, lineage metadata, and packaging behavior. A briefing, digest, or summary surface may travel inside an output package or be referenced by one, but the surface does not replace the package or its metadata contract.

A summary may reference recommendation, rationale, packet, and output-package objects, but it must not replace them. Derived surface convenience must not become object substitution.

## Interaction with Review, Post-Mortem, and Learning

Review briefings may summarize review posture, packet context, or likely resolution issues for a reviewer, but they do not replace review resolution objects, disposition objects, or packet sufficiency rules. A review-oriented surface may help a reviewer orient quickly; it cannot convert that orientation surface into the governing review record.

A post-mortem briefing is not the same thing as a post-mortem object. The post-mortem object preserves governed attribution judgment, evidence-quality position, and learning direction. A post-mortem briefing may present the central attribution view for an audience, but it must keep attribution qualification, evidence quality, and traceability honest.

A learning digest is not the same thing as policy-learning admission. A learning digest may summarize why a case or set of cases appears learning-relevant, but it does not itself satisfy admissibility, comparability, threshold, or governance approval. Learning admission remains controlled by the policy-learning evidence admission and update-threshold standard.

Executive summaries must not silently strip uncertainty, authority, or failure context where materially relevant because those omissions can change downstream interpretation even when the audience is not taking direct action. High-level does not mean consequence-free.

Failure and exception briefings must remain distinguishable from normal operator summaries. The platform must not let a structurally degraded, anomalous, quarantined, or recovery-governed state be narrated as ordinary work merely because the audience prefers short surfaces.

## Canon Placement and Extension Rules

This document belongs in the shared objects layer because it governs the shared meaning of reusable briefing, digest, and summary surfaces as controlled derivation layers above many underlying objects. It does not merely govern delivery transport, role entitlement, or one local reporting surface.

Future summary extensions must be placed according to control role, not convenience. If an extension changes shared summarization meaning, compression rules, omission rules, or lineage rules across domains, it belongs in the shared objects layer or shared core canon as appropriate. If it changes delivery metadata or package transport semantics, it belongs in the output-package or interface canon. If it changes role entitlement or scope-filtering rules, it belongs in the boundary canon. If it changes one domain's local presentation ritual or local reporting contract only, it belongs in the relevant domain contract and must not redefine the shared summarization grammar.

This standard does not authorize local documents to redefine recommendation meaning, rationale meaning, packet meaning, review-resolution meaning, post-mortem meaning, learning-admission meaning, or output-package meaning. Those meanings remain with their controlling standards.

## Failure Modes in Briefing, Digest, and Summary Design

### Executive summary with no uncertainty

The executive surface presents the main recommendation or current position as if it were settled while omitting material uncertainty, contradiction, or insufficient-evidence qualification.

### Operator digest with no action-path clarity

The operator surface compresses the central path so far that the recipient cannot tell what action, wait posture, or non-action route is actually being described.

### Review briefing with no authority context

The reviewer receives a derived surface that summarizes the issue but omits who may review, who may bind, what authority is retained, or what boundary still limits action.

### Failure briefing that hides anomaly state

The exception or failure surface uses ordinary operator wording and suppresses the anomaly, failure, quarantine, or recovery posture that makes the case structurally different.

### Post-mortem summary that collapses attribution

The post-mortem surface compresses recommendation error, execution error, override effect, environmental change, or insufficient evidence into one blurred success-or-failure story.

### Compressed summary that drops traceability

The surface is short and readable, but it no longer preserves stable linkage back to the case, recommendation, rationale, packet, execution, post-mortem, or learning objects from which it was derived.

### Presentation-safe summary that erases material risk

The surface removes enough detail to satisfy delivery constraints, but in doing so it hides material failure, urgency, authority, or uncertainty context that the audience still needs for safe interpretation.

### Digest treated as if it were the underlying object

The platform or a downstream user begins relying on a digest as if it were the recommendation, the rationale trace, the packet, the post-mortem object, or the learning-admission record itself.

### Learning digest reused as if it were already admitted evidence

The surface summarizes apparent learning relevance and is then reused as though comparability, evidence quality, attribution strength, and admission status had already been governed.

## Non-Negotiables

1. A summary surface is not the same thing as a recommendation, and no summary may be treated as recommendation authority by compression alone.
2. A digest is not the same thing as a rationale trace, and no digest may replace the authoritative internal reasoning structure.
3. A human-review packet is not the same thing as a briefing surface, and no briefing may be treated as packet sufficiency by narrative convenience.
4. Presentation-safe briefing is not the same thing as internal full-fidelity briefing, and presentation filtering must not be confused with semantic equivalence.
5. Compression must not become semantic rewrite, even when the audience prefers shorter or cleaner surfaces.
6. Omission risk must remain explicit whenever materially relevant qualifiers or source detail have been compressed away.
7. Summary sufficiency is not the same thing as decision sufficiency, and audience-fit adequacy must never be mistaken for commitment, instruction, review, or learning readiness.
8. Every governed briefing, digest, and summary surface must preserve traceable linkage back to the underlying controlled objects from which it was derived.
9. A post-mortem briefing is not the same thing as a post-mortem object, and a learning digest is not the same thing as policy-learning admission.
10. Future summary extensions must be placed according to control role, not convenience, and domain-local presentation habits must not redefine the shared summarization grammar.

## Closing Statement

If this standard remains intact, the platform can produce executive, operator, review, exception, post-mortem, and learning-facing surfaces that are shorter, safer, and more audience-fit without allowing those surfaces to become semantic substitutes for the underlying canonical objects.

That discipline is necessary if the Fourth Form platform is to remain one governed decision system rather than drifting into summary culture, presentation-led reinterpretation, or untraceable narrative convenience.
