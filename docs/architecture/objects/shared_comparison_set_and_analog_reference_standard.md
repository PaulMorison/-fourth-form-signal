# Shared Comparison Set and Analog Reference Standard for the Fourth Form Retail Decision Intelligence Platform

## Purpose of This Document

This document defines the shared platform standard for comparison sets, analog references, precedent references, and cohort references across all current and future domains.

It exists because the platform now has governed standards for decision cases, decision memory, evidence, rationale, uncertainty, constraints, action paths, review packets, execution outcomes, observation maturity, post-mortem attribution, briefing surfaces, and policy-learning admission, but it still lacks one shared meaning for how the current case may be compared against prior cases or grouped prior episodes without confusing comparison support with benchmark-safe exposure, current-case evidence, recommendation, attribution, or policy-learning admission.

Without a shared standard, the platform will drift into domain-specific analog habits, precedent invoked as though it were automatically binding, cohort patterns treated as case-specific proof, benchmark cohorts silently reused as case-analog sets, comparison language that hides temporal or outcome mismatch, briefing surfaces that compress comparison lineage until it disappears, review driven by anecdotes instead of governed references, and learning reuse that begins from suggestive historical similarity rather than from disciplined admission rules.

This document is therefore a control document for shared comparison-set and comparison-reference structure.

It defines the core concepts, shared object meanings, comparability and reference-quality rules, interaction rules, extension rules, and governance linkage that all domains must follow when constructing, preserving, or reusing historical comparison support around a current case.

It is the canonical shared comparison-set and analog-reference document for the platform. Future domain workflow contracts, review surfaces, post-mortem practice, briefing surfaces, policy-learning review, and other comparison-bearing artifacts must align with it unless a formal decision record explicitly revises it.

## Role of This Document in the Platform

This document governs the shared comparison-reference grammar that sits between preserved decision history on one side and current-case interpretation, review support, post-mortem support, briefing support, and learning review support on the other.

The shared decision case and decision memory standard defines how decision episodes are anchored and remembered, but it does not define one shared meaning for when a remembered case becomes a governed analog, a governed precedent, or a governed cohort reference for a current case. The platform benchmark-safe comparison and cohort construction standard defines how comparative output may be safely exposed and how disclosure-safe cohorts are formed, but it does not define one shared meaning for comparison references inside the decision loop or for whether a prior case is a valid analog for current interpretation. The shared evidence bundle and signal provenance standard defines what counted as decision-time evidence, but it does not define one shared meaning for how prior-case comparison may support interpretation without becoming current-case evidence. The shared review resolution and case disposition standard defines how prior cases resolved, but it does not define one shared meaning for when prior handling becomes a governed precedent reference. The shared observation-horizon and measurement-window standard defines when outcomes are temporally mature enough to compare seriously, but it does not define one shared meaning for a comparison set or for comparison sufficiency. The shared post-mortem and attribution judgment standard defines how later judgment separates recommendation weakness, execution weakness, override effect, and environmental change, but it does not define one shared meaning for how earlier cases may be compared to the current case without replacing attribution judgment. The shared briefing, digest, and summary surface standard defines how comparison-bearing surfaces may be compressed for audiences, but it does not define the underlying shared meaning of the comparison references those surfaces carry. The shared human-review-packet and intervention-handoff standard defines packet sufficiency and accountable handoff, but it does not define one shared meaning for which prior cases belong in a packet as governed analogs, precedents, or cohort references. The policy-learning evidence admission and update-threshold standard defines when preserved history is admissible for learning and when it is strong enough to support policy change, but it does not define one shared meaning for historical comparison support before admission discipline has been satisfied.

In practical terms, this document governs what a comparison set is, what an analog reference is, what a precedent reference is, what a cohort reference is, what makes a comparison reference valid enough for its intended use, how comparison differs from memory, benchmarking, evidence, recommendation, review resolution, and learning admission, and how later decision-loop stages may reuse governed comparisons without silently promoting them into stronger authority than they actually have.

This document therefore governs historical comparison support as part of platform coherence.

## Core Thesis

In the Fourth Form platform, comparison sets, analog references, precedent references, and cohort references must remain first-class governed comparison-support objects whose scope, comparability basis, reference quality, mismatch visibility, purpose, and lineage remain explicit enough that the platform can compare the current case against prior cases or grouped prior episodes without turning historical similarity, prior handling, or grouped patterns into ungoverned proof, ungoverned authority, or ungoverned learning input.

That is the core thesis.

A comparison set is not the same thing as benchmark-safe cohort construction. An analog reference is not the same thing as decision memory by itself. A precedent reference is not the same thing as current-case evidence. A cohort reference is not the same thing as a recommendation. Comparison support is not the same thing as policy-learning admission. Superficial similarity is not enough for governed analog use. Temporal mismatch must remain explicit. Outcome mismatch must remain explicit. Misleading comparison risk must remain explicit. Future comparison extensions must be placed according to control role, not convenience.

## What This Standard Is and Is Not

This standard is the shared platform rule for how the system records, links, preserves, and reuses governed comparison sets and governed comparison references.

It is not a search-feature note. It is not an analytics memo. It is not a benchmarking playbook. It is not a local workflow guide. It is not a substitute for the decision-memory standard. It is not a substitute for the benchmark-safe comparison boundary standard. It is not a substitute for current-case evidence, recommendation, review resolution, post-mortem attribution, or policy-learning admission. It is not permission for domains to select analogs by intuition alone, to treat prior outcomes as current-case proof, to treat precedent as automatically binding, to treat cohort patterning as case-specific action authority, or to compress comparison references into anecdote and later call that governed support.

A real shared comparison-set and analog-reference standard means the platform can answer the following questions for any material decision episode: what governed references were assembled to compare the current case with prior cases or grouped prior episodes, why each reference belonged in that set, what kind of reference each one was, what comparability basis supported it, what mismatch or limitation remained active, whether the reference was useful for interpretation, caution, review, post-mortem, or learning review, whether comparison sufficiency existed for that use, and how later readers can trace the comparison back to the underlying cases, resolutions, outcomes, and maturity conditions without ambiguity.

## Why a Comparison Set and Analog Reference Standard Is Necessary

Domains must not define comparison-set and analog-reference semantics independently because the platform cannot remain one governed decision system if one domain treats any prior similar-looking case as an analog, another treats prior reviewer handling as binding precedent, another treats aggregate cohort patterning as case-specific proof, another silently imports benchmark cohorts into case-comparison logic, and another feeds learning from historical comparison patterns before learning-admission discipline is satisfied.

If comparison-reference grammar is left local, several failures follow. One domain preserves structural comparability while another preserves only surface resemblance. One domain preserves temporal and observation-window mismatch while another hides it inside narrative shorthand. One domain treats prior handling as a precedent only when authority and disposition are actually comparable while another treats any prior resolution as binding by habit. One domain uses cohort references as grouped pattern support while another lets them dictate current-case action. One domain preserves comparison lineage into review packets and briefings while another compresses that lineage away until only anecdote remains. Post-mortem then cannot tell whether comparison helped interpretation or distorted it, review cannot tell whether historical references were serious or casual, and policy learning begins adapting from suggestive comparison patterns that never crossed evidence-admission discipline.

The platform therefore needs one shared standard so that future domains can extend one governed comparison grammar rather than inventing their own local meanings for comparison sets, analogs, precedents, cohorts, comparability, mismatch, and comparison sufficiency.

## Core Concepts

The platform uses the following core concepts.

### Comparison set

Comparison set is the governed set of references used to compare the current case against other cases or episodes.

### Analog reference

Analog reference is a governed prior-case comparison reference where prior case similarity is meaningful enough to inform interpretation, caution, or review.

### Precedent reference

Precedent reference is a governed prior case or prior disposition whose relevance comes from prior handling or prior judgment, not merely similarity.

### Cohort reference

Cohort reference is a governed grouped comparison surface used for pattern comparison across multiple cases.

### Comparison eligibility

Comparison eligibility is the governed judgment that a candidate prior case, prior disposition, or grouped prior set may legitimately enter a comparison set for a stated purpose because scope, lineage, and comparability are strong enough for that use.

### Analog validity

Analog validity is the governed judgment that a prior case is structurally, temporally, and outcome-relevantly similar enough to the current case that analog use is informative rather than misleading.

### Precedent validity

Precedent validity is the governed judgment that a prior case or prior disposition is relevant because its prior handling, authority path, or judgment structure is meaningfully applicable to the current review, even if the prior case is not a close analog on surface facts.

### Cohort comparability

Cohort comparability is the governed judgment that a grouped set of prior cases is similar enough in scope, structure, maturity, and outcome meaning to support pattern comparison without implying case-specific proof.

### Misleading comparison risk

Misleading comparison risk is the governed risk that a comparison reference or comparison set will distort interpretation because surface resemblance, scope drift, maturity mismatch, outcome mismatch, or authority mismatch is being overread.

### Comparison lineage

Comparison lineage is the reconstructible chain connecting the current case, the comparison purpose, the selected comparison set, the comparability basis, the mismatch notes, and the later review, briefing, post-mortem, or learning-review surfaces that reused that set.

### Analog lineage

Analog lineage is the reconstructible chain connecting the current case, the selected prior analog case, the similarity judgment that justified analog use, the mismatch notes that qualified it, and the later surfaces that reused it.

### Precedent lineage

Precedent lineage is the reconstructible chain connecting the current case, the selected prior disposition or prior handling reference, the authority or judgment basis that made it relevant, and the later surfaces that reused it.

### Cohort lineage

Cohort lineage is the reconstructible chain connecting the current case, the grouped prior cases used for pattern comparison, the cohort-construction basis used for that comparison purpose, the comparability and mismatch notes attached to that group, and the later surfaces that reused it.

### Comparison sufficiency

Comparison sufficiency is the governed judgment that the comparison support assembled for a stated purpose is strong enough to aid that purpose honestly without pretending that comparison alone settles the case.

### Comparison-scope discipline

Comparison-scope discipline is the governed rule that a comparison set must remain matched to the authorized decision scope, comparison purpose, and reference class being used, rather than expanding opportunistically to whichever prior cases make the present argument look strongest.

### Structural comparability

Structural comparability is the governed judgment that the current case and the prior reference share materially similar case structure, decision meaning, action-space shape, and constraint profile strongly enough for serious comparison.

### Temporal comparability

Temporal comparability is the governed judgment that the current case and the prior reference sit in sufficiently compatible timing, regime, freshness, and observation-horizon conditions for the intended comparison use.

### Outcome comparability

Outcome comparability is the governed judgment that the outcomes being compared are measured over sufficiently compatible horizons, maturity states, and commercial meanings for the intended comparison use.

## Shared Comparison Set

At platform level, a shared comparison set is the formal governed set of references used to compare the current case against other cases or episodes.

It exists because the platform must preserve more than that historical cases were looked at. It must preserve what current case the comparison served, what comparison purpose governed selection, which references were included, which references were excluded, what comparison-scope discipline constrained the set, what comparability and mismatch basis justified the set, and what later review, briefing, post-mortem, or learning-review surfaces depended on it.

The shared comparison set must preserve, conceptually, all of the following. It must preserve a comparison-set ID so the governed set has stable identity. It must preserve the current-case reference so the comparison remains anchored to the present decision episode. It must preserve a comparison-purpose reference so later systems can distinguish review support, caution support, precedent review, cohort patterning, post-mortem comparison, or learning-review support. It must preserve included reference IDs and excluded-reference notes where relevant so later systems can reconstruct why the set looked the way it did. It must preserve comparison-scope discipline, comparison eligibility, comparison sufficiency, misleading comparison risk, and comparison lineage so the set remains inspectable rather than rhetorical. It must preserve lineage or version reference and timestamp so later systems can reconstruct which governed set existed at the relevant time.

The shared comparison set is therefore not a casual reading list and not a raw search result. It is the governed reference set above memory retrieval and below later comparison-bearing surfaces.

## Shared Analog Reference

At platform level, a shared analog reference is a governed prior-case comparison reference where prior case similarity is meaningful enough to inform interpretation, caution, or review.

It exists because the platform must preserve more than that two cases looked similar. It must preserve why the prior case counts as a serious analog, what kind of similarity actually mattered, which structural, temporal, scope, or outcome mismatches remained active, and whether analog validity was strong enough for interpretation, caution, review orientation, or post-mortem comparison.

The shared analog reference must preserve, conceptually, all of the following. It must preserve an analog-reference ID so the governed reference has stable identity. It must preserve current-case reference and prior-case reference so the comparison remains reconstructible. It must preserve analog validity, structural comparability, temporal comparability, outcome comparability, and misleading comparison risk. It must preserve mismatch notes where relevant so superficial closeness does not hide deeper non-comparability. It must preserve analog lineage, comparison-purpose reference, and lineage or version reference so later systems can reconstruct why this prior case was treated as informative.

An analog reference is informative when prior-case similarity is meaningful enough to sharpen interpretation, caution, or review. An analog reference is misleading when the prior case is being overread because its resemblance is shallow, its authority path differs materially, its outcome horizon is not comparable, or its environment changed too much for the analogy to hold. Superficial similarity is not enough for governed analog use.

## Shared Precedent Reference

At platform level, a shared precedent reference is a governed prior case or prior disposition whose relevance comes from prior handling or prior judgment, not merely similarity.

It exists because some historical references matter not because they are close analogs on facts, but because the platform or accountable humans previously handled a materially related class of issue in a way that current review, disposition, or escalation must take seriously.

The shared precedent reference must preserve, conceptually, all of the following. It must preserve a precedent-reference ID so the governed reference has stable identity. It must preserve current-case reference and prior-case or prior-disposition reference so the precedent remains reconstructible. It must preserve precedent validity, prior handling or prior judgment basis, authority comparability, disposition comparability, and precedent lineage. It must preserve mismatch notes where relevant so later systems can see when precedent is informative but not binding, or relevant but not factually close. It must preserve comparison-purpose reference and lineage or version reference so later systems can reconstruct why prior handling was treated as relevant here.

Precedent relevance comes from prior handling or prior judgment, not merely similarity. A precedent may matter even if it is not a close analog. A prior high-authority resolution may still inform current review posture, escalation discipline, or closure reasoning even when the prior facts are not tightly analogous. Precedent treated honestly therefore depends on authority comparability and judgment comparability, not just on visual similarity.

## Shared Cohort Reference

At platform level, a shared cohort reference is a governed grouped comparison surface used for pattern comparison across multiple cases.

It exists because some historical comparison is group-based rather than single-case based. The platform must preserve when grouped prior cases are being used to show patterning, baseline range, outcome distribution, or repeated handling tendency without pretending that the grouped pattern itself settles the current case.

The shared cohort reference must preserve, conceptually, all of the following. It must preserve a cohort-reference ID so the governed reference has stable identity. It must preserve current-case reference and grouped prior-case references or grouped prior-case criteria so the cohort remains reconstructible. It must preserve cohort comparability, scope comparability, temporal comparability, outcome comparability, observation-horizon comparability, misleading comparison risk, and cohort lineage. It must preserve grouping rationale, exclusions where relevant, and comparison-purpose reference so later systems can reconstruct why the cohort was built and what it was meant to support. It must preserve lineage or version reference and timestamp so later systems can reconstruct which governed cohort reference existed at the relevant time.

A cohort reference may be useful for patterning but weak for case-specific action. Grouped history can reveal whether the current case looks unusual, ordinary, or risky relative to a repeated set of episodes. It does not therefore prove that the current case should take the path the cohort pattern appears to favor.

## Comparability and Reference-Quality Rules

Comparison references may be included in governed handling only when their reference quality is explicit enough that later readers can tell why the comparison is informative and what limitations still remain.

### Structural comparability

Structural comparability asks whether the current case and the prior reference share materially similar case shape, action-space shape, decision meaning, constraint pattern, and review significance. A prior case with similar language but a materially different action space, materially different constraint bundle, or materially different review posture is not a strong analog merely because it looks familiar.

### Temporal comparability

Temporal comparability asks whether the current case and the prior reference sit inside sufficiently compatible timing conditions, freshness conditions, regime conditions, and state-readiness conditions for the intended use. Temporal mismatch must remain explicit. A case from a materially different regime, materially different decision horizon, or materially different maturity context may still be worth remembering, but it cannot be treated as temporally comparable without qualification.

### Outcome comparability

Outcome comparability asks whether the outcomes being compared were measured over sufficiently similar horizons, with sufficiently similar maturity posture, that the resulting comparison is honest. Outcome mismatch must remain explicit. A superficially favorable prior outcome is not a serious analog if the prior outcome was observed over an incompatible or immature window relative to the current case.

### Scope comparability

Scope comparability asks whether the current case and the prior reference concern sufficiently similar decision populations, operating populations, tenant or client boundaries, and case scopes for the intended use. Comparison-scope discipline requires that scope drift remain visible rather than being smoothed away because a broader or narrower prior case tells a more convenient story.

### Authority comparability

Authority comparability asks whether the prior handling or prior disposition came from a materially relevant authority layer for the current use. This matters especially for precedent references. A prior resolution taken under materially different authority conditions may still be historically interesting, but it is weak precedent if the current case sits under different retained authority, different approval burden, or different review layer.

### Observation-horizon comparability

Observation-horizon comparability asks whether the compared cases were observed over compatible horizons and comparable maturity bases. A prior case with immature or extended observation may still be informative for bounded caution, but it is weak support for outcome comparison or learning review if the current case expects mature comparison.

### Comparison eligibility

Comparison eligibility exists only when the reference class, comparison purpose, scope, maturity, and lineage are explicit enough that the candidate reference belongs inside governed comparison rather than casual historical mention.

### Comparison sufficiency

Comparison sufficiency is purpose-relative. A thin analog may be sufficient to justify caution or to orient a reviewer, but insufficient to support strong precedent reasoning, post-mortem comparison, or learning review. Comparison sufficiency is not the same thing as evidence sufficiency, decision sufficiency, or learning-admission sufficiency.

### Informative versus misleading comparison

Comparison is informative when the comparison purpose is explicit, the reference class is explicit, comparability is explicit, mismatch is explicit, and the comparison is being used for the kind of support it can honestly carry. Comparison is misleading when the reference is selected because it looks compelling rather than because it is genuinely comparable, when mismatch is hidden, when the wrong reference class is being used for the task, or when historical comparison is being treated as if it already settled current-case facts.

Why a superficially similar case may still be invalid as an analog is straightforward. Surface resemblance does not prove shared decision structure, shared timing, shared constraint burden, shared authority context, shared outcome meaning, or shared observation maturity. A prior case can look similar and still fail analog validity badly.

Why a cohort may be useful for patterning but weak for case-specific action is equally straightforward. Grouped cases can show whether the current case sits inside or outside a historical pattern, but a grouped pattern does not preserve all the case-specific state, rationale, uncertainty, constraint, and authority detail needed to justify a present action path.

Why a precedent may matter even if it is not a close analog is that precedent relevance comes from prior handling or prior judgment, not merely similarity. A prior high-authority disposition may govern what kinds of review, escalation, caution, or closure posture are materially relevant now even when the current facts are not tightly analogous.

## Interaction with Benchmarking, Memory, Review, and Learning

A comparison set is not the same thing as benchmark-safe cohort construction. Benchmark-safe cohort construction governs what comparative output may be safely exposed and how disclosure-safe cohorts are built. A shared comparison set governs what historical references are used inside current-case interpretation, review, post-mortem, briefing, or learning review.

An analog reference is not the same thing as decision memory by itself. Decision memory preserves reusable historical episodes. Analog reference is the governed current-case act of selecting one preserved episode as meaningfully similar enough to inform interpretation, caution, or review.

A precedent reference is not the same thing as current-case evidence. Prior handling can matter for review posture, authority awareness, or caution, but it does not by itself prove what is true in the current case.

A cohort reference is not the same thing as a recommendation. Grouped historical patterning may support orientation, caution, or escalation awareness. It does not itself choose the current action path.

Comparison support is not the same thing as policy-learning admission. Historical comparison may help frame a learning question or indicate that repeated pattern review is warranted, but policy-learning evidence admission and update-threshold discipline still control whether those histories become admitted learning evidence.

Post-mortem may use comparison sets, but comparison does not replace attribution judgment. Historical analogs, precedents, and cohorts may sharpen the interpretation of what happened, but the shared post-mortem and attribution judgment standard still governs whether the present case reflects recommendation weakness, execution weakness, override effect, environmental change, information weakness, or insufficient evidence.

Review packets and briefings may include analogs, but those analogs must remain governed references rather than freeform anecdotes. The human-review-packet standard still governs packet sufficiency, and the briefing standard still governs summarization. Comparison lineage must survive into those surfaces strongly enough that later readers can tell what the reference actually was and why it was included.

Current-case to prior-case comparison may be governed only when the comparison purpose, comparison scope, reference class, comparability basis, and mismatch notes are explicit. Comparison may support review, post-mortem, briefing, and learning without silently becoming evidence admission by itself. Learning review is one governed use of that support; policy-learning admission remains separate.

## Canon Placement and Extension Rules

This document belongs in the shared objects layer because it governs the shared meaning of reusable comparison references and grouped comparison surfaces above preserved decision memory and beneath later review, briefing, post-mortem, and learning-review use.

Future comparison extensions must be placed according to control role, not convenience. If an extension changes shared comparison meaning, reference classes, comparability rules, mismatch handling, or lineage rules across domains, it belongs in the shared objects layer or shared core canon as appropriate. If it changes disclosure safety, aggregation safety, or exposure entitlement, it belongs in the boundary canon. If it changes packet assembly, briefing compression, or output transport only, it belongs in the packet, briefing, or interface canon as appropriate. If it changes only one domain's local workflow habit, it belongs in the relevant domain contract and must not redefine the shared comparison grammar.

This standard does not authorize local documents to redefine decision memory, benchmark-safe comparison, current-case evidence, recommendation, review resolution, post-mortem attribution, or policy-learning admission. Those meanings remain with their controlling standards.

## Failure Modes in Comparison and Analog Design

### Analog selected by surface resemblance only

The prior case is chosen because it looks narratively similar while structural comparability, temporal comparability, or outcome comparability remain weak.

### Precedent treated as binding when it is not

Prior handling is invoked as though it automatically governs the present case even though authority comparability, scope comparability, or review-layer comparability are weak.

### Cohort used as if it were case-specific proof

Grouped historical patterning is treated as though it directly proves the correct action path for the current case.

### Temporal mismatch hidden

The comparison surface fails to show that the prior case sits in a materially different timing, regime, freshness, or maturity posture.

### Outcome mismatch hidden

The comparison surface fails to show that the outcomes being compared were observed over materially different horizons or maturity states.

### Comparison set assembled without scope discipline

Historical references are gathered opportunistically from different scopes, populations, or case classes until the set supports the preferred story rather than the honest comparison purpose.

### Review driven by anecdotes instead of governed references

Human reviewers receive remembered stories or freeform claims about prior cases rather than explicit governed analogs, precedents, or cohort references with lineage.

### Learning fed from comparison patterns without admission discipline

Repeated historical patterning is treated as though it already justifies policy adaptation even though learning-admission and update-threshold rules were never satisfied.

### Benchmark cohort silently reused as analog set

An exposure-safe or disclosure-safe benchmark cohort is imported into current-case comparison as though it were already a governed analog or cohort reference for case interpretation.

### Comparison lineage lost in briefing compression

Historical references appear in briefing, digest, or summary surfaces after their source cases, mismatch notes, and reference-class distinctions have been compressed away.

## Non-Negotiables

1. Every governed comparison set must preserve the current-case anchor, explicit comparison purpose, and comparison-scope discipline.
2. Superficial similarity is not enough for governed analog use, and analog validity must be explicit.
3. Temporal mismatch must remain explicit whenever timing, freshness, regime, or maturity differs materially across compared cases.
4. Outcome mismatch must remain explicit whenever outcome horizon, maturity, or commercial meaning differs materially across compared cases.
5. Precedent relevance must remain distinct from current-case evidence and must never be treated as automatically binding merely because prior handling exists.
6. A cohort reference may support pattern comparison, but it must never be treated as case-specific proof or as a recommendation by itself.
7. Comparison support may inform review, briefing, post-mortem, and learning review, but comparison support is not the same thing as policy-learning admission.
8. A comparison set is not the same thing as benchmark-safe cohort construction, and benchmark cohorts must not be silently reused as analog or precedent sets.
9. Comparison lineage, analog lineage, precedent lineage, and cohort lineage must remain traceable through packets, briefings, post-mortems, and memory reuse.
10. Future comparison extensions must be placed according to control role, not convenience, and domain-local comparison habits must not redefine the shared comparison grammar.

## Closing Statement

The platform needs governed comparison because historical reference is powerful and easy to misuse. When analogs, precedents, and cohort references are controlled as first-class shared objects, the platform can use prior history to sharpen interpretation, caution, review, post-mortem, and learning review without confusing comparison with evidence, recommendation, attribution, or policy admission. That separation is what keeps comparison useful without letting it become quietly authoritative.
